# Motor Control Architecture (Rover)

This document describes the motor control architecture for a 6-wheel chassis
using L298N drivers on an ATmega2560.

## 1. Motor Abstraction: `SixWheelRover`

`SixWheelRover` groups 6 individual `Motor` instances for coordinated
differential (tank) drive. Each motor type is independent via generics:

```rust
pub struct SixWheelRover<M1, M2, M3, M4, M5, M6> {
    pub frontal_right: M1,
    pub frontal_left:  M2,
    pub center_right:  M3,
    pub center_left:   M4,
    pub rear_right:    M5,
    pub rear_left:     M6,
}
```

For a homogeneous array (required by `RoverController`), use `ErasedMotor`.
See `docs/consideration_implementation.md` for the full rationale.

## 2. PWM Timer Assignment (Current Implementation)

Hardware PWM is generated by the ATmega2560's internal timers. The current
assignment uses three separate timers — one per L298N driver:

| Driver | Timer | PWM Pins | Direction Pins |
| :--- | :--- | :--- | :--- |
| **Driver 1 (Front)** | Timer 2 | D10 (OC2A), D9 (OC2B) | D22–D25 |
| **Driver 2 (Center)** | Timer 3 | D5 (OC3A), D2 (OC3B) | D26–D29 |
| **Driver 3 (Rear)** | Timer 4 | D6 (OC4A), D7 (OC4B) | D30–D33 |

> **⚠️ Pin conflict — D2 (PE4):** D2 is shared between OC3B (Timer3 motor
> PWM) and INT4 (encoder interrupt). They cannot be active at the same time.
> If an encoder must use INT4, reassign the center-left motor to OC3C (D3)
> and accept losing INT5 there, or route that encoder to a PCINT pin instead.

> **Timer 0** is reserved by `arduino-hal` for `delay_ms` / `delay_us`.
> Do NOT assign any motor to Timer 0.

## 3. Hall Effect Encoders

Encoders detect motor rotation via magnetic pulses. External interrupts are
used to avoid missing pulses at high rotation speed.

### Encoder Trait

```rust
pub trait Encoder {
    fn get_counts(&self) -> i32;
    fn reset(&self);
}
```

`HallEncoder` uses an AVR `Mutex<Cell<i32>>` for interrupt-safe access from
both the main loop and ISR context.

### Interrupt Pin Assignment

| Motor | Pin | Hardware Interrupt | Conflict |
| :--- | :--- | :--- | :--- |
| Front Right  | D21 | INT0 (PD0) | None |
| Front Left   | D20 | INT1 (PD1) | None |
| Center Right | D19 | INT2 (PD2) | Shared with USART1 RX (Serial1 / RPi link) |
| Center Left  | D18 | INT3 (PD3) | Shared with USART1 TX (Serial1 / RPi link) |
| Rear Right   | D2  | INT4 (PE4) | Shared with Timer3 OC3B (center-left motor PWM) |
| Rear Left    | D3  | INT5 (PE5) | None |

> **D18 / D19** are the USART1 lines used to communicate with the RPi.
> Using INT2/INT3 for encoders on those pins requires moving the RPi link
> to a different USART (e.g. Serial2 on D16/D17).

## 4. Software Requirements

For bare-metal interrupt handling in Rust:

1. Enable feature: `#![feature(abi_avr_interrupt)]`
2. Dependency: `avr-device` for interrupt macros
3. Enable global interrupts: `unsafe { avr_device::interrupt::enable() }`
