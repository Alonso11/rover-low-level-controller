# Documentación ACS712-30A — Sensor de Corriente por Efecto Hall

## 1. Descripción General

El ACS712ELCTR-30A es un sensor de corriente de efecto Hall con salida analógica.
Mide corriente AC o DC de hasta ±30 A y genera un voltaje proporcional a la corriente
medida, con el punto cero centrado en VCC/2 = 2.5 V.

Se usa en el rover para monitorear la corriente de los motores u otras cargas.

## 2. Especificaciones Eléctricas

| Parámetro               | Valor              |
| :---------------------- | :----------------- |
| Voltaje de alimentación | 5 V                |
| Rango de corriente      | ±30 A              |
| Sensibilidad            | **66 mV/A**        |
| V_out @ 0 A             | VCC/2 = **2500 mV** |
| V_out @ +30 A           | 2500 + 30×66 = 4480 mV |
| V_out @ −30 A           | 2500 − 30×66 = 520 mV  |
| Impedancia dinámica     | < 1 Ω              |
| Tipo de salida          | Analógico (0 – 5 V)|

> ⚠️ La sensibilidad de 66 mV/A es exclusiva de la versión **30 A**.
> Las versiones 5 A (185 mV/A) y 20 A (100 mV/A) tienen valores distintos.

## 3. Conexión al Arduino Mega

Se usan **6 módulos ACS712-30A**, uno por motor. Cada OUT conectado a su canal ADC:

| Motor         | Pin Arduino Mega | Registro |
| :------------ | :--------------- | :------- |
| Front Right   | **A0** (PF0)     | ADC0     |
| Front Left    | **A1** (PF1)     | ADC1     |
| Center Right  | **A2** (PF2)     | ADC2     |
| Center Left   | **A3** (PF3)     | ADC3     |
| Rear Right    | **A4** (PF4)     | ADC4     |
| Rear Left     | **A5** (PF5)     | ADC5     |

VCC → 5V y GND → GND en todos los módulos.

## 4. Fórmula de Conversión

El ADC del ATmega2560 es de 10 bits con referencia AVCC = 5 V:

```
V_mv       = (adc * 5000) / 1023      // voltaje en mV
delta_mv   = V_mv − 2500              // desviación respecto al cero
I_mA       = (delta_mv × 1000) / 66  // corriente en mA (puede ser negativa)
```

Ejemplo a 10 A:
- V_out = 2500 + 10 × 66 = 3160 mV
- ADC ≈ (3160 × 1023) / 5000 ≈ 647
- delta = 3160 − 2500 = 660 mV → I = 660 × 1000 / 66 = **10 000 mA = 10 A**

## 5. Driver — `src/sensors/acs712.rs`

El driver es puro Rust sin dependencias de `arduino_hal`. Recibe el valor ADC
crudo para que la lógica de conversión sea testeable en x86.

```rust
let acs712 = ACS712::new();                    // offset estándar 2500 mV
let acs712 = ACS712::with_zero_mv(2487);       // offset calibrado

let current_ma = acs712.read_ma(adc_raw);      // i32, puede ser negativo
let over = acs712.is_overcurrent(current_ma, 2000); // true si |I| > 2000 mA (FAULT)
```

El manejo del ADC hardware (pin + `analog_read`) queda en `main.rs`.

## 6. Calibración del Offset

El punto cero teórico es 2500 mV, pero puede variar ±20 mV según la unidad.
Con el canal desconectado (0 A real), leer el ADC crudo del ejemplo y calcular:

```
zero_mv_real = (adc_0A * 5000) / 1023
```

Pasar ese valor a `ACS712::with_zero_mv(zero_mv_real)` en `main.rs`.

## 7. ADC Multiplexado y Promediado

El ATmega2560 tiene **un solo ADC de 10 bits** con multiplexor de 16 canales.
Solo convierte un canal a la vez (~104 µs por conversión a 125 kHz).

En `main.rs` se promedia cada canal con la macro `adc_avg!`:

```rust
// 8 muestras × 104 µs = ~832 µs por canal
let raw = adc_avg!(acs_fr_pin, adc, SEN_SAMPLES);
let current_ma = acs712.read_ma(raw);
```

Con 8 muestras el ruido ADC baja de ±2-3 cuentas a ~±1 cuenta (±5 mV),
reduciendo el ruido en corriente a **±75 mA** frente a ±225 mA sin promediado.

Los 6 canales se leen cada `SEN_READ_PERIOD = 25` ciclos (~500 ms).
Tiempo total bloqueante: 6 canales × 8 muestras × 104 µs ≈ **5 ms cada 500 ms**.

## 8. Integración en `main.rs` — Protección Graduada

El bloque de sensores analógicos usa dos velocidades de muestreo independientes:

**Fast tier** (cada 3 ciclos, ~60 ms, 2 muestras/canal):
- Solo detecta FAULT (≥ `OVERCURRENT_FAULT_MA = 2000 mA`)
- ~1.25 ms bloqueantes por ejecución
- Responde a cortocircuitos en ~60 ms

**Slow tier** (cada 25 ciclos, ~500 ms, 8 muestras/canal):
- Clasifica Warn (≥ 1200 mA) / Limit (≥ 1600 mA) / Fault (≥ 2000 mA)
- Actualiza `sensor_frame` para el frame TLM
- ~5.8 ms bloqueantes por ejecución

| Constante | Valor | Fracción L298N (2A) | Acción |
|-----------|-------|---------------------|--------|
| `OVERCURRENT_WARN_MA` | 1200 mA | 60 % | Log en TLM, rover sigue |
| `OVERCURRENT_LIMIT_MA` | 1600 mA | 80 % | Velocidad limitada al 60 % |
| `OVERCURRENT_FAULT_MA` | 2000 mA | 100 % | Para todo, espera RST |

Las corrientes se incluyen en el frame TLM (campos I0–I5), no en frames SEN separados.
Ver `docs/consideration_implementation.md` §8 y §9 para el diseño completo.
